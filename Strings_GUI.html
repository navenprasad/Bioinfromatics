4.4 The StringTest project

Get the project stringtest from the subject web site.

It has in it two classes:

StringTester:

    This is the main class that has a user interface. Create an object of this class to start the application. It will show a graphical user interface that offers some string operations. This class is complete you do not have to change it.

StringEngine:

    This is the class that implements the string operation functionality. This class is not properly implemented. It is your job to implement the operations in this class.

The interface of the application shows a row of buttons at the top, a string in the middle (the "display"), and a text field and some buttons at the bottom. The display initially says "Welcome to the String Tester".

    The buttons are intended to do the following:

UPPERCASE: Switches the display to upper case.

lowercase: Switches the display to lower case.

Titlecase: Switches the display to title case. Initially, just try to turn the first letter of the display to upper case, and the rest to lower case. As a real challenge task (after you finish everything else), try to start each word with an upper case letter.

Strip: Remove all spaces at the beginning and the end of the display.

Length: Display the length (number of characters) of the display.

Quit: Quit the application.

Append: Add the text from the text filed to the end of the display.

Prepend: Add the text from the text filed to the start of the display string.

Insert: Insert the string in the text field into the middle of the displayed string.

Replace: Replace the display with the string in the text field.

The interface for this application is already implemented. You only need to implement the StringEngine class. Most of the method signatures are already written,

Hint:

To implement the StringEngine methods, you need to get access to the display string and the string from the text field (the "input string"). You can get them from the StringTester class: The string tester has two methods:

    public String getDisplayString()

and

    public String getInputString()

that return those strings. To get access to these methods, you need a reference to the StringTester object in the StringEngine object. To achieve this, do the following:

  • define an instance field named "userInterface" in the StringEngine class;

  • add a parameter "interfaceObject" to the constructor of StringEngine;

  • in the constructor, assign "interfaceObject" to "userInterface";

  • in the StringTester class, where the StringEngine object is created (and the constructor called) you need to add the parameter. Pass in "this" as the parameter that will pass the StringTester object itself;

  • in the StringEngine methods implementing the operations, you can now write lines such as
    String display = userInterface.getDisplayString();
    String input = userInterface.getInputString();

Finally, add a JTextField outputText; to the StringTester GUI, just under the JLabel. Re-write the output method so that output is displayed in both the JLabel and the JTextField.

Another important hint:

Don't work alone! Work in pairs or small groups, walk around, look at other peoples ideas, discuss!

5.3 GUI for StringTester

Copy the StringTester code from the previous week's directory into this week's. Look at StringTester.java - note that each button has a separate ActionListener, which is in a separate class. Re-write StringTester.java so that the StringTester class handles all of the button events itself. That is, it must become an ActionListener by implementing the ActionListener interface. Each time you create a button, add 'this' as its ActionListener. Then StringTester.java must have an 'actionPerformed(ActionEvent event)' method to handle all of the button events.
If you want, finally add a JLabel with your favorite (small) image to the StringTester GUI.

5.4 Make a Java project

After re-writing the StringTester source code, combine StringTester.java and StringEngine.java into a Java project. To do this, put both files into a subdirectory of the current one, named, e.g., mySTT. In each file insert the line 'package mySTT;' at the very top of the file. Remember, the package declaration must be the first statement in the file. I find that both BlueJ and JCreator handle packages in ways that are difficult to understand. So, from the command line, in the directory ABOVE mySTT, to compile them type
'javac mySTT/StringTester.java' .
To run them type, again in the parent directory,
'java mySTT.StringTester'
(Note that since they were created in BlueJ, they may not have a 'main(String[] args)' method, in which case you will have to write one in the StringTester.java file.
